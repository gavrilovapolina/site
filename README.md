# Лендинг polinagavrilova.ru — публикация

Сайт — один файл `index.html`, без сборки и зависимостей.
Фото, og-картинка (1200×630), favicon — готовы в `assets/`. CNAME-файл для GitHub Pages создан.

## Шаг 1. Аккаунт GitHub (делает Полина, ~5 минут)

1. Открыть github.com → Sign up.
2. Ввести почту, придумать пароль и логин (например, `polinagavrilova`).
3. Подтвердить почту кодом из письма.

## Шаг 2. Деплой (делает Claude после появления аккаунта)

1. Установить GitHub CLI: `winget install GitHub.cli`
2. `gh auth login` — Полина один раз вводит код в браузере.
3. Создать публичный репозиторий `site`, запушить содержимое `landing/`.
4. Включить Pages (branch main) + custom domain `polinagavrilova.ru`.

## Шаг 3. DNS в Тильде (делает Полина по готовым записям)

Домен куплен у Тильды → Тильда сама держит DNS.
В личном кабинете Tilda: Сайты → Домены (или настройки домена) → «Управление DNS» / «DNS-записи».

Добавить:

| Тип | Хост | Значение |
|-----|------|----------|
| A | @ | 185.199.108.153 |
| A | @ | 185.199.109.153 |
| A | @ | 185.199.110.153 |
| A | @ | 185.199.111.153 |
| CNAME | www | `<логин>.github.io.` |

Удалить старые A-записи Тильды (если домен был привязан к её сайту).

## Шаг 4. После привязки

1. В GitHub Pages дождаться проверки домена, включить «Enforce HTTPS» (до часа).
2. Проверить: открытие сайта, все 9 ссылок кейсов, превью ссылки в Telegram, мобильную версию.

## Правки

Тексты правятся прямо в `index.html` — все секции подписаны комментариями
(Hero, About, PM Projects, Design Projects, Useful, Contact).
